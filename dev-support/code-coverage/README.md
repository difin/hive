# Static code analysis

## Running static code analysis locally

The `run-coverage.sh` script in the folder `/dev-support/code-coverage` runs maven build and tests with the profile `coverage`
which generates the test coverage data for the java classes by using the jacoco-maven-plugin.

If the required parameters are given it also runs the Sonar code analysis and uploads the results to the given SonarQube Server.

### Prerequisites
- Have the JDK installed for Hive. Always check, it's Java 8 (AdoptOpenJDK: 1.8.0.292) at time of writing this readme.
- If you want to use Sonar, either configure jenv and check the java versions (look for blocks calling `jenv global ...`) or change the script to use the solution we use in the Jenkins build script (see section **Notes** below). Sonar plugin needs Java 11/17.

### Running the analysis

After running the script, the reports generated by the Jacoco
can be found under the folders `/target/jacoco-aggregate` of
the related modules.

Here is how you can generate the code coverage report:

1. cd into `dev-support/code-coverage`
2. run `./run-coverage.sh -s <YOUR_MAVEN_SETTINGS_FILE>`
3. locate and open the **index.html** in the folder `/target/jacoco-aggregate` of the module that you want to see the coverage data.

### Publishing the coverage results to SonarQube

To publish the results to our SonarQube application, there are some required parameters:
- Sonar host URL (-u),
- project key (-k)
- login credentials / token (-t),

Here is an example command for running code coverage analysis and publishing the coverage data in SonarQube:

```
cd dev-tools/code-coverage
./run-coverage.sh -n Hive -u https://sonarqube.infra.cloudera.com/ -k hive-cdh-main -t <SONAR_TOKEN> -s ~/work/downstream/.m2/settings.xml
```

The token can be generated at https://sonarqube.infra.cloudera.com/account/security

- select **generate token**
- enter a token name: e.g. hive-cdw-coverage-report
- select type: project analysis token
- select the Sonar project, or create one if you need a new, e.g: Hive CDW
- set expiration: no expiration
- save the token, you won't be able to retrieve it later

### Notes

As the Sonar maven plugin requires Java version higher than 8, you can do this:
- configure jenv to have both Java 8 and Java 11 or 17 available on your machine. The script `dev-support/code-coverage/run-coverage.sh` tries to use jenv.
- or download and extract java 11 or 17 (temurin) and modify the script to set JAVA_HOME environment variable to point to this java version just before calling maven with the profile **sonar**. The `hive-sonar.Jenkinsfile` in the repo `hive-test-kube` uses this method.  

## Running static code analysis with Jenkins

You can start a static code analysis manually. To do this:

Open the Jenkins page for hive manual precommit builds:
https://hiveci.rke-us-west-04.kc.cloudera.com/job/internal-hive-precommit-manual/

You need to be logged in and have the rights to configure the build. Here, you can overwrite the Jenkins build script with our build script prepared for code coverage analysis and Sonar reporting.
Locate the file `hive-sonar.Jenkinsfile` under the folder `job-dsl` in our build repository:
https://github.infra.cloudera.com/Kitchen/hive-test-kube

Set the repo, git branch and sonar parameters in the script. See the configs for cdh_main and cdw-master.

```
def REPO_URL="https://github.infra.cloudera.com/CDH/hive.git"

def GIT_BRANCH="cdw-master"
def SONAR_URL="https://sonarqube.infra.cloudera.com/"
def SONAR_PROJECT_KEY="hive-cdw-master"
def SONAR_TOKEN="sqp_0d5969037235068019e973bcdc1973a440dcf637"
```

```
def REPO_URL="https://github.infra.cloudera.com/CDH/hive.git"

def GIT_BRANCH="cdh_main"
def SONAR_URL="https://sonarqube.infra.cloudera.com/"
def SONAR_PROJECT_KEY="hive-cdh-main"
def SONAR_TOKEN="sqp_a8ec6a22144fae77a36b502009d4244a241b0420"
```

Run the build

Find the Sonar reports at:
* https://sonarqube.infra.cloudera.com/dashboard?id=hive-cdw-master
* https://sonarqube.infra.cloudera.com/dashboard?id=hive-cdh-main

**Don't forget to restore the build script to the original one!**
You can do it right after your build is started.
See the original script here: https://github.infra.cloudera.com/Kitchen/hive-test-kube/blob/rke2/job-dsl/hive.Jenkinsfile
